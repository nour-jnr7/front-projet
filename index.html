Structure d'une règle CSS

Une règle CSS est composée d'un sélecteur et d'un bloc de déclarations. Comprendre cette structure est fondamental pour maîtriser CSS.

Anatomie d'une règle CSS

Définition : Une règle CSS est une instruction qui applique un style à un ou plusieurs éléments HTML. Elle se compose de deux parties principales : le sélecteur (qui cible les éléments) et le bloc de déclarations (qui définit les styles).

/* Structure de base */
sélecteur {
    propriété: valeur;
    propriété: valeur;
}
Exemple simple 
/* Cible tous les paragraphes */
p {
    color: blue;        /* Déclaration 1 */
    font-size: 16px;    /* Déclaration 2 */
    margin: 20px;       /* Déclaration 3 */
}

Ceci est un paragraphe stylisé avec les règles ci-dessus.

Exemple avec multiples sélecteurs 
/* Cible h1, h2 et h3 en même temps */
h1, h2, h3 {
    font-family: Arial, sans-serif;
    color: #333;
    margin-bottom: 1rem;
}

Composants d'une déclaration

  • Propriété : le nom de la caractéristique à modifier (ex : color, font-size)
  • Valeur : la nouvelle valeur assignée (ex : red, 16px)
  • Deux-points : sépare la propriété de la valeur
  • Point-virgule : termine chaque déclaration (obligatoire sauf pour la dernière, mais recommandé)

Unités CSS communes

Unité Type Description Exemple
px Absolue Pixels font-size: 16px;
% Relative Pourcentage du parent width: 50%;
em Relative Relatif à la taille de police parent margin: 1.5em;
rem Relative Relatif à la taille de police root padding: 2rem;
vh/vw Relative Viewport height/width height: 100vh;

!important et spécificité

Le mot-clé !important force une déclaration à avoir la priorité maximale. À éviter sauf cas exceptionnel car il rend le code difficile à maintenir.

/* Utilisation (non recommandée) */
.button {
    background: red !important; /* Surcharge toutes autres règles */
}
Bonnes pratiques
  • Toujours terminer chaque déclaration par un point-virgule
  • Indenter le code pour une meilleure lisibilité
  • Grouper les propriétés connexes (positionnement, box-model, typographie, visuel)
  • Utiliser des commentaires pour les sections complexes
  • Préférer les unités relatives (rem, em) pour la scalabilité
  • Éviter !important - privilégier une meilleure spécificité
Erreurs courantes
  • Oublier le point-virgule entre déclarations
  • Utiliser un espace au lieu de deux-points (color blue au lieu de color: blue)
  • Abuser de !important au lieu de résoudre les problèmes de spécificité
  • Mélanger les guillemets dans les valeurs

Types de sélecteurs CSS

Les sélecteurs permettent de cibler précisément les éléments HTML à styliser. Maîtriser les différents types est crucial pour écrire du CSS efficace.

Sélecteur universel

Syntaxe : *

Cible tous les éléments du document. Utilisation courante pour les resets CSS.

* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}

Performance : Éviter pour des règles complexes, peut être coûteux.

Sélecteur de type (élément)

Syntaxe : element

Cible tous les éléments d'un type donné.

p { color: navy; }
div { display: block; }
a { text-decoration: none; }

Ce paragraphe utilise le sélecteur p.

Sélecteur de classe

Syntaxe : .classname

Cible tous les éléments avec l'attribut class="classname". Le plus utilisé en pratique.

.btn { 
    padding: 10px 20px; 
    background: blue;
    color: white;
}

.btn-large { font-size: 18px; }

Bonne pratique : Utiliser des noms de classes descriptifs et réutilisables (BEM : Block-Element-Modifier).

Sélecteur d'ID

Syntaxe : #idname

Cible l'élément unique avec l'attribut id="idname". Haute spécificité.

#header { 
    background: #f0f0f0; 
    height: 80px;
}

#main-content { max-width: 1200px; }

Attention : Les ID ont une spécificité très élevée, difficile à surcharger. Préférer les classes pour le styling.

Sélecteur d'attribut

Cible les éléments selon leurs attributs HTML.

/* Présence d'attribut */
[disabled] { opacity: 0.5; }

/* Valeur exacte */
[type="text"] { border: 1px solid gray; }

/* Commence par */
[href^="https"] { color: green; }

/* Termine par */
[src$=".pdf"] { 
    background: url(pdf-icon.svg);
}

/* Contient */
[class*="btn"] { cursor: pointer; }

Cas d'usage : Styliser des formulaires, liens externes, types de fichiers.

Pseudo-classes

Syntaxe : :pseudo-classe

Cible les éléments dans un état spécifique ou selon leur position.

/* États interactifs */
a:hover { color: red; }
a:active { transform: scale(0.95); }
input:focus { border-color: blue; }
button:disabled { opacity: 0.5; }

/* États de validation */
input:valid { border-color: green; }
input:invalid { border-color: red; }

/* Position dans le parent */
li:first-child { margin-top: 0; }
li:last-child { margin-bottom: 0; }
li:nth-child(odd) { background: #f9f9f9; }
li:nth-child(3n) { color: blue; }

/* Autres */
:not(.special) { color: gray; }
:root { --main-color: blue; }
Survolez ce lien

Accessibilité : Toujours accompagner :hover de :focus-visible pour le clavier.

Pseudo-éléments

Syntaxe moderne : ::pseudo-element (double deux-points)

Crée des éléments virtuels pour styliser des parties spécifiques.

/* Avant et après */
.quote::before {
    content: "« ";
    color: gray;
}
.quote::after {
    content: " »";
    color: gray;
}

/* Première ligne/lettre */
p::first-line { font-weight: bold; }
p::first-letter { 
    font-size: 2em; 
    float: left;
}

/* Sélection */
::selection {
    background: yellow;
    color: black;
}

Note : Ancienne syntaxe :before encore supportée mais ::before est recommandé.

Spécificité des sélecteurs

Lorsque plusieurs règles ciblent le même élément, la spécificité détermine laquelle s'applique. Calcul en 4 niveaux : (inline, IDs, classes/attributs/pseudo-classes, éléments/pseudo-éléments)

Sélecteur Spécificité Priorité
p (0, 0, 0, 1) Faible
.btn (0, 0, 1, 0) Moyenne
#submit (0, 1, 0, 0) Haute
style="" (inline) (1, 0, 0, 0) Très haute
!important ∞ Maximum
Exemple de conflit de spécificité
/* Spécificité (0,0,1,0) */
.btn { color: blue; }

/* Spécificité (0,1,0,0) - GAGNE */
#submit { color: red; }

/* Même si .btn vient après, #submit gagne */

Bonne pratique : Garder une spécificité basse et homogène. Éviter les ID pour le styling, préférer les classes.

Combinateurs CSS

Les combinateurs permettent de créer des relations entre sélecteurs pour cibler des éléments selon leur position dans le DOM.

Descendant (espace)

Syntaxe : A B

Cible tous les B descendants de A (à n'importe quel niveau).

/* Tous les 
 dans .container */
.container p {
    line-height: 1.6;
}

/* Tous les  dans 
 */
nav a {
    text-decoration: none;
}

Enfant direct (>)

Syntaxe : A > B

Cible uniquement les B enfants directs de A.

/* Uniquement les 
  •  directs de 
       */
ul > li {
    list-style: square;
}

/* Enfants directs de .card */
.card > h3 {
    margin-top: 0;

    Syntaxe : A + B

    Cible B immédiatement après A (même parent, position adjacente).

    /* 
     juste après 
     */

    font-weight: bold;
    margin-top: 0;

/* Label après input checkbox */
input[type="checkbox"] + label {
    cursor: pointer;

    Frère général (~)

    Syntaxe : A ~ B

    Cible tous les B frères après A (même parent, pas forcément adjacents).

    /* Tous les 
     après 
     */
h2 ~ p {
    color: navy;
}

/* Inputs après un input:checked */
input:checked ~ input {
    opacity: 0.5;
}

    Combinaisons avancées

    On peut chaîner plusieurs combinateurs pour des sélections très précises.

    /* Tous les  dans 
  •  enfants directs de 
     */
nav > ul > li a {
    padding: 10px;
}

/* 
     adjacent à 
     dans .article */
.article h2 + p {
    font-size: 1.1em;
}

/* Tous les  après  dans .form */
.form input:focus ~ span {
    color: blue;
}
    • Erreurs courantes
    • Confondre + (adjacent immédiat) et ~ (tous les suivants)
    • Oublier que + et ~ ne ciblent que les frères suivants, jamais les précédents
    • Surqualifier les sélecteurs : div.container > div.row au lieu de .container > .row
    • Trop de niveaux de descendance : impacte performance et spécificité
    Bonnes pratiques
    • Préférer > à l'espace quand possible (plus performant, plus explicite)
    • Limiter la profondeur des sélecteurs (max 3-4 niveaux)
    • Utiliser des classes plutôt que de longues chaînes de combinateurs
    • Documenter les sélecteurs complexes avec des commentaires

    Couleurs et mise en forme

    CSS offre plusieurs formats de couleurs et propriétés de mise en forme. Maîtriser ces outils est essentiel pour créer des interfaces attractives et accessibles.

    Formats de couleurs CSS

    1. Noms de couleurs

    140 noms prédéfinis (ex : red, blue, lightgray)

    color: red;
background: cornflowerblue;
border-color: transparent;

    2. Hexadécimal

    Syntaxe : #RRGGBB ou #RGB (raccourci)

    color: #ff0000;        /* Rouge */
color: #f00;           /* Rouge (raccourci) */
color: #336699;        /* Bleu gris */
color: #369;           /* Bleu gris (raccourci) */
color: #00000080;      /* Noir 50% opacité (8 digits) */

    3. RGB / RGBA

    Syntaxe : rgb(red, green, blue) ou rgba(red, green, blue, alpha)

    Valeurs : 0-255 pour RGB, 0-1 pour alpha

    color: rgb(255, 0, 0);           /* Rouge */
color: rgb(51, 102, 153);        /* Bleu gris */
background: rgba(0, 0, 0, 0.5);  /* Noir semi-transparent */
background: rgba(255, 255, 255, 0.9); /* Blanc presque opaque */

    4. HSL / HSLA

    Syntaxe : hsl(hue, saturation, lightness)

    • Hue : teinte en degrés (0-360) - 0=rouge, 120=vert, 240=bleu
    • Saturation : pourcentage (0%=gris, 100%=couleur pure)
    • Lightness : luminosité (0%=noir, 50%=normal, 100%=blanc)
    color: hsl(0, 100%, 50%);        /* Rouge pur */
color: hsl(120, 100%, 50%);      /* Vert pur */
color: hsl(240, 100%, 50%);      /* Bleu pur */
background: hsl(200, 50%, 70%);  /* Bleu clair */
background: hsla(0, 0%, 0%, 0.3); /* Noir transparent */

    Avantage : Plus intuitif pour créer des variations (éclaircir/assombrir en changeant L, désaturer en changeant S)

    5. Variables CSS (Custom Properties)

    Définir des couleurs réutilisables dans :root ou n'importe quel sélecteur.

    :root {
    --primary-color: #6366f1;
    --secondary-color: #8b5cf6;
    --text-dark: #1a1a2e;
    --bg-light: #f8f9fa;
}

.button {
    background: var(--primary-color);
    color: white;
}

.card {
    background: var(--bg-light);
    color: var(--text-dark);
}

    Bonne pratique : Utiliser des variables pour faciliter la maintenance et le theming.

    Propriétés principales

    Propriété Description Exemple
    color Couleur du texte color: #333;
    background-color Couleur de fond background-color: lightblue;
    border-color Couleur de bordure border-color: red;
    opacity Opacité globale (0-1) opacity: 0.5;
    box-shadow Ombre portée box-shadow: 0 4px 10px rgba(0,0,0,0.2);
    text-shadow Ombre de texte text-shadow: 2px 2px 4px gray;

    Dégradés (Gradients)

    Dégradé linéaire

    /* Simple */
background: linear-gradient(to right, red, blue);

/* Avec angle */
background: linear-gradient(45deg, #667eea, #764ba2);

/* Multiples couleurs */
background: linear-gradient(to bottom, 
    red 0%, 
    yellow 50%, 
    green 100%
);

    Dégradé radial

    background: radial-gradient(circle, white, black);
background: radial-gradient(ellipse at center, 
    rgba(255,255,255,1) 0%, 
    rgba(0,0,0,1) 100%
);

    Accessibilité et contraste

    Le contraste entre texte et fond est crucial pour l'accessibilité (WCAG).

    Ratios de contraste minimums

    • Niveau AA : 4.5:1 pour texte normal, 3:1 pour texte large
    • Niveau AAA : 7:1 pour texte normal, 4.5:1 pour texte large
    Exemples de contrastes
    Bon contraste - #333 sur blanc (12.6:1) ✓
    Bon contraste - Blanc sur #6366f1 (5.8:1) ✓
    Mauvais contraste - #ccc sur blanc (1.8:1) ✗
    Outils de vérification
    • WebAIM Contrast Checker - Vérifier les ratios
    • Chrome DevTools - Indicateur de contraste dans l'inspecteur
    • Axe DevTools - Extension pour audits a11y
    Bonnes pratiques couleurs
    • Ne jamais utiliser la couleur comme seul moyen de transmettre l'information
    • Tester avec des simulateurs de daltonisme
    • Utiliser des variables CSS pour centraliser la palette
    • Préférer HSL pour créer des variations cohérentes
    • Vérifier les contrastes sur tous les états (hover, focus, disabled)
    • Éviter les couleurs pures très saturées pour de grandes surfaces

    Erreurs courantes à éviter

    1

    Surqualification des sélecteurs

    Erreur : div.container > div.row > div.col

    Mieux : .container > .row > .col

    Les balises ajoutent de la spécificité inutile et rendent le code moins flexible.

    2

    Abus de !important

    Erreur : Utiliser !important pour résoudre tous les conflits

    Mieux : Comprendre et gérer la spécificité correctement

    !important rend le code impossible à surcharger et crée des dettes techniques.

    3

    Oublier le point-virgule

    Erreur : color: red background: blue;

    Mieux : color: red; background: blue;

    Sans point-virgule, les déclarations suivantes sont ignorées.

    4

    Sélecteurs trop profonds

    Erreur : .header .nav .menu .item .link (5 niveaux)

    Mieux : .nav-link (classe directe)

    Impact négatif sur performance et maintenabilité.

    5

    Mauvais contraste de couleurs

    Erreur : Texte gris clair sur fond blanc

    Mieux : Vérifier les ratios de contraste (4.5:1 minimum)

    Problèmes d'accessibilité pour les utilisateurs malvoyants.

    6

    Mauvaise utilisation de :not()

    Erreur : *:not(.special):not(.other):not(.more)

    Mieux : Utiliser des classes positives

    Complexité élevée et performance dégradée.

    7

    Unités inappropriées

    Erreur : Utiliser px partout pour les tailles de police

    Mieux : Utiliser rem ou em pour la scalabilité

    Les unités relatives permettent un meilleur responsive et accessibilité.

    8

    Oublier :focus-visible

    Erreur : Styliser uniquement :hover

    Mieux : Toujours inclure :focus-visible

    Essentiel pour la navigation au clavier et l'accessibilité.

    Mini FAQ

    Quelle est la différence entre classe et ID ?

    Les classes (.nom) peuvent être réutilisées sur plusieurs éléments et ont une spécificité moyenne. Les ID (#nom) doivent être uniques sur la page et ont une spécificité très élevée. Pour le styling, privilégiez toujours les classes.

    Quand utiliser HSL plutôt que RGB ?

    HSL est plus intuitif pour créer des variations de couleur. Pour éclaircir une couleur, il suffit d'augmenter la luminosité (L). Pour la désaturer, réduire la saturation (S). RGB est plus direct pour reproduire des couleurs exactes, mais moins pratique pour les variations.

    Comment résoudre un conflit de spécificité sans !important ?

    Plusieurs solutions : 1) Augmenter la spécificité avec une classe supplémentaire, 2) Réorganiser l'ordre des règles dans le CSS, 3) Utiliser un sélecteur plus spécifique, 4) Dupliquer le sélecteur (.btn.btn a plus de spécificité que .btn).

    Pourquoi mes styles ne s'appliquent-ils pas ?

    Vérifiez : 1) La spécificité (une règle plus spécifique l'emporte), 2) L'ordre dans le fichier CSS (la dernière règle gagne à spécificité égale), 3) La syntaxe (point-virgule, deux-points), 4) Le sélecteur (correspond-il vraiment à l'élément ?), 5) Le cache du navigateur.

    Que faire si j'ai besoin de cibler un élément très spécifique ?

    PrivilContinuer20:59&égier l'ajout d'une classe unique plutôt que d'empiler des combinateurs. Par exemple, au lieu de .header > .nav > .menu > li > a.active, créez simplement une classe .nav-link-active. C'est plus maintenable et performant.
